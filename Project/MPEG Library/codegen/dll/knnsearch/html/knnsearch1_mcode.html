1function [idx, dist] = knnsearch(X,Y,varargin)
  2%KNNSEARCH Find K nearest neighbors.
  3%   IDX = KNNSEARCH(X,Y) finds the nearest neighbor in X for each point in
  4%   Y. X is an MX-by-N matrix and Y is an MY-by-N matrix. Rows of X and Y
  5%   correspond to observations and columns correspond to variables. IDX is
  6%   a column vector with MY rows. Each row in IDX contains the index of
  7%   the nearest neighbor in X for the corresponding row in Y.
  8%
  9%   [IDX, D] = KNNSEARCH(X,Y) returns a MY-by-1 vector D containing the
 10%   distances between each row of Y and its closest point in X.
 11%
 12%   [IDX, D]= KNNSEARCH(X,Y,'NAME1',VALUE1,...,'NAMEN',VALUEN) specifies
 13%   optional argument name/value pairs:
 14%
 15%     Name          Value
 16%     'K'           A positive integer, K, specifying the number of nearest
 17%                   neighbors in X to find for each point in Y. Default is
 18%                   1. IDX and D are MY-by-K matrices. D sorts the
 19%                   distances in each row in ascending order. Each row in
 20%                   IDX contains the indices of K closest neighbors in X
 21%                   corresponding to the K smallest distances in D.
 22%
 23%     'NSMethod'    Nearest neighbors search method. Value is either:
 24%                   'kdtree'    - Creates and uses a kd-tree to find
 25%                                 nearest neighbors. 'kdtree' is only valid
 26%                                 when the distance metric is one of the
 27%                                 following metrics:
 28%                                   - 'euclidean'
 29%                                   - 'cityblock'
 30%                                   - 'minkowski'
 31%                                   - 'chebyshev'
 32%                   'exhaustive' - Uses the exhaustive search algorithm.
 33%                                  The distance values from all the points
 34%                                  in X to each point in Y are computed to
 35%                                  find nearest neighbors.
 36%                   Default is 'kdtree' when the number of columns of X is
 37%                   not greater than 10, X is not sparse, and the distance
 38%                   metric is one of the above 4 metrics; otherwise,
 39%                   default is 'exhaustive'.
 40%
 41%    'IncludeTies'  A logical value indicating whether KNNSEARCH will
 42%                   include all the neighbors whose distance values are
 43%                   equal to the Kth smallest distance. Default is false.
 44%                   If the value is true, KNNSEARCH includes all these
 45%                   neighbors. In this case, IDX and D are MY-by-1 cell
 46%                   arrays. Each row in IDX and D contains a vector with at
 47%                   least K numeric numbers. D sorts the distances in each
 48%                   vector in ascending order. Each row in IDX contains the
 49%                   indices of the closest neighbors corresponding to these
 50%                   smallest distances in D.
 51%
 52%    'Distance'     A string or a function handle specifying the distance
 53%                   metric. The value can be one of the following:
 54%                   'euclidean'   - Euclidean distance (default).
 55%                   'seuclidean'  - Standardized Euclidean distance. Each
 56%                                   coordinate difference between X and a
 57%                                   query point is scaled by dividing by a
 58%                                   scale value S. The default value of S
 59%                                   is the standard deviation computed from
 60%                                   X, S=NANSTD(X). To specify another
 61%                                   value for S, use the 'Scale' argument.
 62%                   'cityblock'   - City Block distance.
 63%                   'chebychev'   - Chebychev distance (maximum coordinate
 64%                                   difference).
 65%                   'minkowski'   - Minkowski distance. The default
 66%                                   exponent is 2. To specify a different
 67%                                   exponent, use the 'P' argument.
 68%                   'mahalanobis' - Mahalanobis distance, computed using a
 69%                                   positive definite covariance matrix C.
 70%                                   The default value of C is the sample
 71%                                   covariance matrix of X, as computed by
 72%                                   NANCOV(X). To specify another value for
 73%                                   C, use the 'Cov' argument.
 74%                   'cosine'      - One minus the cosine of the included
 75%                                   angle between observations (treated as
 76%                                   vectors).
 77%                   'correlation' - One minus the sample linear
 78%                                   correlation between observations
 79%                                   (treated as sequences of values).
 80%                   'spearman'    - One minus the sample Spearman's rank
 81%                                   correlation between observations
 82%                                  (treated as sequences of values).
 83%                   'hamming'     - Hamming distance, percentage of
 84%                                   coordinates that differ.
 85%                   'jaccard'     - One minus the Jaccard coefficient, the
 86%                                   percentage of nonzero coordinates that
 87%                                   differ.
 88%                   function      - A distance function specified using @
 89%                                   (for example @DISTFUN). A distance
 90%                                   function must be of the form
 91%  
 92%                                   function D2 = DISTFUN(ZI, ZJ),
 93%  
 94%                                   taking as arguments a 1-by-N vector ZI
 95%                                   containing a single row of X or Y, an
 96%                                   M2-by-N matrix ZJ containing multiple
 97%                                   rows of X or Y, and returning an
 98%                                   M2-by-1 vector of distances D2, whose
 99%                                   Jth element is the distance between the
100%                                   observations ZI and ZJ(J,:).
101%
102%    'P'            A positive scalar indicating the exponent of Minkowski
103%                   distance. This argument is only valid when 'Distance'
104%                   is 'minkowski'. Default is 2.
105%  
106%    'Cov'          A positive definite matrix indicating the covariance
107%                   matrix when computing the Mahalanobis distance. This
108%                   argument is only valid when 'Distance' is
109%                  'mahalanobis'. Default is NANCOV(X).
110%  
111%    'Scale'        A vector S containing non-negative values, with length
112%                   equal to the number of columns in X. Each coordinate
113%                   difference between X and a query point is scaled by the
114%                   corresponding element of S. This argument is only valid
115%                   when 'Distance' is 'seuclidean'. Default is NANSTD(X).
116%  
117%    'BucketSize'   The maximum number of data points in the leaf node of the
118%                   kd-tree (default is 50). This argument is only
119%                   meaningful when kd-tree is used for finding nearest
120%                   neighbors.
121% 
122%   Example:
123%      % Find 2 nearest neighbors in X and the corresponding values to each
124%      % point in Y using the distance metric 'cityblock'
125%      X = randn(100,5);
126%      Y = randn(25, 5);
127%      [idx, dist] = knnsearch(X,Y,'dist','cityblock','k',2);
128%
129%   See also CREATENS, ExhaustiveSearcher, KDTreeSearcher, RANGESEARCH.
130
131%   Copyright 2009 The MathWorks, Inc.
132
133
134
135pnames = { 'k' 'nsmethod' 'bucketsize', 'includeties'};
136dflts =  { 1   []         []             false};
137[numNN, nsmethod, bSize, includeTies, ~,args] =...
138     internal.stats.parseArgs(pnames, dflts, varargin{:});
139
140O=createns(X,args{:},'nsmethod', nsmethod,'bucketSize',bSize);
141if nargout < 2
142    idx = knnsearch(O,Y,'k',numNN, 'includeties', includeTies);
143else
144    [idx, dist] = knnsearch(O,Y,'k',numNN, 'includeties',includeTies);
145end
146end
